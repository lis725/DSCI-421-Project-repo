# DSCI 421 Final Project: GPU-Accelerated Parasitic Capacitance Prediction

## Overview

This project implements a GPU-accelerated deep learning surrogate model for parasitic capacitance prediction from BEM geometry files. The pipeline parses BEM polygon layouts, rasterizes geometry into 7-channel 256x256 tensors, fuses raster features with four design parameters, and predicts capacitance values with a ResNet18-based multi-head regression model.

The project was tested on the ECE HPC platform with two NVIDIA GeForce RTX 2080 Ti GPUs. Results include CPU vs single-GPU training, CPU vs GPU inference, dual-GPU DataParallel experiments, cached-data optimization, AMP inference/training, and an ablation study.

## Directory Guide

- `pattern_res/`: BEM geometry inputs and ground-truth capacitance tables for type1/type2/type3.
- `ckpt/`: provided trained checkpoint used for inference and evaluation.
- `eval_out/`: generated by `step16_eval_viz.py` if the evaluation command is rerun; the submission package stores curated evaluation outputs in `report_results/` to stay under the 50 MB upload limit.
- `submit_out/`: generated prediction CSV/TXT files and submission file.
- `report_results/`: curated report-ready metrics, plots, environment logs, and experiment summary.
- `step10_dataset_raster.py`: dataset parsing and rasterization smoke test.
- `step13_train_fused_params.py`: main CPU/GPU/multi-GPU training script.
- `step14_infer_export.py`: inference/export script.
- `step16_eval_viz.py`: evaluation and visualization script.
- `step17_optimized_experiment.py`: cached-data and AMP optimization experiment.
- `step18_inference_benchmark.py`: CPU/GPU/AMP/multi-GPU inference benchmark.
- `step19_ablation_study.py`: params-only, raster-only, and fused-model ablation study.
- `step20_cpu_gpu_consistency_check.py`: CPU-vs-GPU prediction consistency check on a few samples.
- `parallel_utils.py`: optional multi-GPU DataParallel helpers.
- `project_paths.py`: local relative path configuration.


Expected final benchmark from the report run:

```text
cpu fp32: time=606.4667s samples=28800 throughput=47.49 samples/s
cuda fp32: time=26.6587s samples=28800 throughput=1080.32 samples/s
cuda amp: time=22.9371s samples=28800 throughput=1255.61 samples/s
cuda:all fp32: time=26.1736s samples=28800 throughput=1100.34 samples/s
```

Interpretation: GPU inference is much faster than CPU inference. Dual-GPU inference provides little additional speedup over a single GPU for this small dataset and lightweight model because DataParallel communication overhead dominates.


Expected final summary:

```text
params_only: time=2.80s best_val=0.000057 mae_t1=0.004593 mae_t2=0.006661 mae_t3=0.005727
raster_only: time=5.38s best_val=0.000649 mae_t1=0.012193 mae_t2=0.033668 mae_t3=0.018426

```

Interpretation: in this small dataset, the four low-dimensional design parameters are highly predictive. The raster geometry representation is more general, but the ResNet-based model may need more data/tuning to outperform the parameter-only model.

## CPU/GPU Consistency Check

Run this after confirming CUDA is available:

```bash
~/mamba/envs/dsci421_gpu/bin/python step20_cpu_gpu_consistency_check.py --num_samples 5
```

Expected behavior: the script loads the same checkpoint on CPU and GPU, runs the same samples, and prints maximum and mean absolute prediction differences. Small differences are expected because CPU and GPU kernels are not bit-exact.
